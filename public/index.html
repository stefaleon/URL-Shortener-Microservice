URL Shortener Microservice

  • User Story: I can pass a URL as a parameter and I will receive a shortened URL in the JSON response.
  • User Story: If I pass an invalid URL that doesn't follow the valid http://www.example.com format, the JSON response will contain an error instead.
  • User Story: When I visit that shortened URL, it will redirect me to my original link.

    • Usage

    The /new/SOMEURL route adds a new shortUrl to the db if SOMEURL is a valid url.

    Example:
    http://localhost:3000/new/https://nodejs.org/en/
    produces an output like the following:
    {"original_url":"https://nodejs.org/en/","short_url":"http://localhost:3000/1008"}

    Now http://localhost:3000/1008 redirects to https://nodejs.org/en/.
